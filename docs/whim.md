# Whim: 「思いつき」を自律的に実現する

mywant の核心は **「やりたいことを書いておくだけでいい」** という体験です。
今すぐ実現できなくても、何から始めればいいかわからなくても、Whim に書いた意図は消えずに生き続け、条件が整った瞬間に動き始めます。

---

## 1. ユーザーストーリー: 「シリコンバレーに移住する」

### 始まり: 成田空港での思いつき
エンジニアの田中さんは、出張帰りに「いつか家族でシリコンバレーに住みたい」と思い、mywant に一行書きます。

```yaml
wants:
  - metadata: { name: "シリコンバレー移住", type: whim }
    spec:
      params:
        want: "家族4人でシリコンバレーに移住したい。子供の入学前の3年以内が理想。"
```

### 霧を晴らす「分解」
Thinker エージェントが起動し、目標を達成するための「地図」を提案します。
1. **調査**: ビザの種類、現地企業の採用プロセス、学区の調査。
2. **待機**: 調査結果が出るまで、他の Want は draft 状態で待機します。

### 「壁」を「ステップ」に変える
調査の結果、「O-1A ビザ」の申請には実績不足であることが判明します。
システムはアラートを出す代わりに、**実績を積むための新しい Want**（技術ブログ執筆、OSS貢献、登壇準備）をツリーに追加します。

### Capability アンロックによる加速
1年後、田中さんの OSS 活動が実を結び `visa_application_support` 能力がアンロックされます。
待機中だった「ビザ申請書作成」Want が即座に `executing` に切り替わり、自動生成が始まります。

### 達成: パロアルトの朝
3年後、田中さんはパロアルトのアパートで Whim の達成通知を受け取ります。
52個の Want と 341件のタスクが、3年かけて現実を動かし続けました。

## 2. 技術設計: Goal Thinker (自律プランニング)

Whim を実現するのは、システムの現在の能力（Capability）に基づいてプランを立てる **Goal Thinker** という脳です。

### アーキテクチャ
Goal Thinker は MCP (Model Context Protocol) サーバーを通じて、以下の手札を自ら発見（Discovery）します。
- **Available Wants**: `knowledge`, `reminder`, `itinerary` 等、今すぐ使える Want 型。
- **Capabilities**: API アクセス権限や外部連携。
- **Achievements**: ユーザーが過去に達成した実績。

### 拡張されたステートマシン
1. **Analyzing (Discovery)**: MCP からシステムの手札を取得し、LLM と共に現状を把握。
2. **Questioning**: 具体化のために情報が足りない場合、ユーザーに質問を提示。
3. **Decomposing (Planning)**: 手札の中から最適な Want 型を組み合わせてツリーを生成。
4. **Monitoring & Evolution**: 新しい実績の解除や子 Want の完了を検知した瞬間に **再プランニング** を実行し、ツリーをさらに深く伸ばします。

### 増殖サイクルの連鎖
- **最初**: `knowledge` しかない → 「ビザの調査」を生成。
- **中盤**: 調査完了により `visa_expert` 実績が解除。
- **進化**: Thinker が新実績を発見 → 自動的に「ビザ申請書作成」を親 Goal の下に生やす。

---

## 3. Whim が提供する価値

- **「書いた瞬間から価値がある」**: 意図が記録され、システムが「いつか実行できる状態」を待ち続けます。
- **「分解は自分でしなくていい」**: Thinker が「どう砕くか」を提案し、あなたは選択するだけです。
- **「達成が次の可能性を開く」**: 小さな達成がスキルツリーのように Capability をアンロックし、より大きな目標を実現可能にします。
