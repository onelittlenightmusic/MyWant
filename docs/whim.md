# mywant でこんなことができるようになります

## Whim（思いつき）から現実へ

mywant の核心は **「やりたいことを書いておくだけでいい」** という体験です。

今すぐ実現できなくても構いません。何から始めればいいかわからなくても構いません。
Whim に書いた意図は消えずに生き続け、条件が整った瞬間に動き始めます。

---

## ストーリー: 「家族でシリコンバレーに移住する」

### 始まり: サンフランシスコ出張の帰り道

**田中さん（38歳、エンジニア）、成田空港に降り立った瞬間に思う。**

> 「あっちで暮らしたい。子供が小学校に上がる前に、家族でシリコンバレーに住む経験をさせたい。
> でも何から始めればいいか、全くわからない。」

その夜、mywant を開いて一行書く。

```yaml
wants:
  - metadata:
      name: 家族でシリコンバレーに移住する
      type: whim
    spec:
      params:
        want: "家族4人（自分・妻・子供2人）でシリコンバレーに移住したい。
               子供が小学校に上がる前の3年以内が理想。
               仕事は現地就職かリモート継続かまだ決めていない。"
```

```
状態: reaching（到達中）
達成条件: 未解析
担当 Thinker: 起動待ち
```

それだけ。田中さんは寝た。

---

### 翌朝: Thinker の最初の問い

> **[Thinker]** 「シリコンバレー移住」を分解するために、まずいくつか教えてください。
>
> **仕事はどうしますか？**
> - A) 今の会社でリモート継続
> - B) 現地の Tech 企業への就職を目指す
> - C) まだ決めていない（両方調べる）
>
> **子供の教育は？**
> - A) 現地の公立校（英語環境）
> - B) 日本人学校
> - C) インターナショナルスクール

田中さんは「B（現地就職）」「A（現地公立）」を選ぶ。

するとWhimの下に最初の地図が現れる。

```
家族でシリコンバレーに移住する (whim)
  ├─ [調査] ビザの種類を特定する
  ├─ [調査] 現地 Tech 企業の採用プロセスを理解する
  ├─ [調査] 公立小学校の入学条件・学区を調べる
  ├─ [調査] 生活費・住居費の相場（Bay Area）
  └─ [待機] その他（上記の結果次第）
```

すべて「draft」状態。まだ何も動いていない。でも**霧が少し晴れた。**

---

### 1ヶ月後: 調査フェーズが自走する

田中さんが日常業務をこなしている間、調査 Want たちが静かに動いている。

> **[MyWant] 調査完了: ビザの種類**
>
> あなたのケースに該当する可能性が高いビザ:
> - **H-1B**（スポンサー企業が必要・抽選あり）
> - **O-1A**（卓越した能力の証明で申請可・抽選なし）
> - **EB-1A**（永住権・高いハードル）
>
> 現地就職を目指す場合、H-1B の抽選リスクを避けるなら
> **O-1A の申請要件を並行して調査します**（自動で子 Want を追加）

田中さんは何もしていないのに、地図が更新されていく。

```
家族でシリコンバレーに移住する (whim)　　reaching → 10%
  ├─ ビザの種類を特定する            ✅ completed
  │   └─ O-1A ビザの要件調査         🔄 in_progress（自動追加）
  ├─ 現地 Tech 企業の採用プロセス    ✅ completed
  │   結果: LeetCode・システム設計・行動面接の3段階が標準
  ├─ 公立小学校の入学条件            ✅ completed
  │   結果: 学区内居住が必須。Cupertino・Palo Alto 学区が人気
  ├─ 生活費・住居費の相場            ✅ completed
  │   結果: 4人家族で月100〜150万円（エリアによる）
  └─ 税務・社会保険の整理            🔄 in_progress
```

---

### 3ヶ月後: 「壁」が現れる

> **[MyWant] アラート: ブロッカー検出**
>
> **O-1A ビザ** の申請には以下の実績が必要です:
> - 業界への顕著な貢献（OSS・論文・登壇）
> - 高水準の報酬の証明
> - 著名な組織との協働実績
>
> 現在の田中さんの実績では**申請にやや不足しています。**
> → 実績を積むための Want を追加しますか？

これが `reaching` の本質。**今はできない。でも何をすればできるかが見える。**

田中さんは「はい」を選ぶ。

```
家族でシリコンバレーに移住する (whim)
  └─ [実績積み上げ] ← 新フェーズ追加
       ├─ 技術ブログを英語で月2本書く       📅 定期実行
       ├─ Google I/O・AWS re:Invent への登壇  ⏳ CFP 監視中
       ├─ OSS へのコントリビューション        🔄 GitHub 連携済み
       └─ Tech 系カンファレンスの CFP を自動検出・通知
```

ゲームで言えば**「隠し条件が出た」**瞬間。普通なら心が折れる。でも mywant はそれを次の Want に変える。

---

### 並走: 面接準備も自走する

実績を積みながら、採用面接の準備も別の Want として動き始める。

```
  ├─ [面接準備] ← 並走フェーズ
       ├─ LeetCode 問題を週5問スケジュール    📅 定期実行
       ├─ ターゲット企業リサーチ              🔄 monitoring
       │   └─ Google・Stripe・Figma を監視中
       │       条件: Senior Engineer・Visa スポンサーあり
       └─ 英語面接の模擬練習                  📅 週2回スケジュール済み
```

田中さんがすることは**練習するだけ**。スケジュール管理はシステムがやる。

```

  ├─ 面接準備                  ✅ completed
  ├─ Visa 申請書類の準備        🔄 in_progress（自動生成中）
```

田中さんだけでなく、**家族全員の準備が並走している。**

## Whim が実現する体験

### 「書いた瞬間から価値がある」

Whim はまだ実行できなくても存在することに意味があります。
意図が記録され、システムが「いつか実行できる状態」を待ち続けます。

### 「分解は自分でしなくていい」

Thinker が「どう砕くか」を提案します。
あなたがすることは **選択だけ**。手順の設計はしなくていい。

### 「壁が次のステップになる」

「今はまだできない」という障壁を検知したら、
mywant はそれを新しい Want に変えます。詰まった瞬間が、前進の起点になります。

### 「達成が次の可能性を開く」

小さな達成 → Capability アンロック → より大きな Want が実行可能になる。
ゲームのスキルツリーのように、現実が広がっていきます。

### 「待つことができる」

実行できない Whim は消えずに `reaching` で待ちます。
「今はまだできないが、いつかやりたいこと」がシステムの中に生き続けます。

---

## Whim がなければ

| 課題 | 従来 |
|---|---|
| 複雑すぎて全体が見えない | Excelで管理 → 更新が止まる |
| 依存関係が把握できない | タスク管理アプリ → 依存・自動監視がない |
| 途中で諦める | モチベーションが続かない |
| 家族全員の準備を並走できない | 自分のことで手一杯 |

## Whim があると

> **「3年後の自分がどこにいるか」を、今日書いておくだけでいい。**
>
> あとはシステムが、できることから少しずつ、現実を動かし続ける。
